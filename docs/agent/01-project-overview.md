# 项目概览

## 项目名称

`ArcMind`

中文定位：弧核智能终端。

命名含义：`Arc` 指环形能量核心、粒子轨道和桌面终端的视觉识别；`Mind` 指大模型对话、记忆和私人助手智能内核。

## 当前阶段

当前仓库处于工程基线整理阶段，已建立 AGENT 协作系统，并已初始化 Electron / React / Three.js 桌面原型。

当前原型能力：

- Electron main process 可创建桌面窗口，并通过 preload 暴露应用版本读取能力。
- Renderer 已有 React 对话界面、输入框、麦克风开关、静音开关和设置入口占位。
- Three.js `ParticleCore` 已作为全屏粒子视觉背景运行，并能响应麦克风音量与高层视觉状态。
- 视觉系统已支持 token 脉冲、错误闪断、思考聚合、语音波形、启动动画和音频频段响应。
- `src/shared` 已开始承载聊天消息、会话状态、视觉状态、AI 流式事件和应用错误等共享类型。
- Vitest 已作为测试基础接入，用于覆盖共享状态转换和轻量 renderer 逻辑。
- AI Runtime 已注入 ArcMind 人格 prompt，并支持用户手动管理、启用和删除的长期记忆。
- 产品化基础已包含 renderer 错误边界、main process 脱敏日志、renderer 崩溃一次性恢复、运行版本信息和 Electron smoke test。

第一版默认方向：

- Windows 桌面应用优先。
- 沉浸式粒子视觉作为第一屏核心体验。
- 接入云端大模型 API。
- 支持文字聊天、点击说话、语音播报。
- 本地保存会话历史、设置和用户可管理的长期记忆。
- 暂不做系统级电脑控制、唤醒词、云同步账号和插件市场。

## 项目目标

ArcMind 目标是成为一个本地运行的 AI 对话终端，让用户以自然语言和 AI 助手交流，并获得类似私人智能管家的沉浸式反馈。

第一版聚焦：

- 可用的流式聊天体验。
- 状态驱动的粒子视觉：待机、聆听、思考、回答、错误、静音。
- 可选语音输入和语音播报。
- 可配置的大模型供应商和模型参数。
- 本地可查看、可删除的历史与记忆。
- 桌面软件基本能力：设置、错误提示、打包和运行日志。

## Agent 开发目标

本仓库要求 agent 以长期维护项目的方式工作：

- 先理解项目，再修改代码。
- 先确认模块边界，再跨模块变更。
- 先复用现有模式，再新增抽象。
- 先验证结果，再汇报完成。
- 发现规则缺口时，推动文档体系演进。

## 非目标

- v1 不实现持续麦克风监听和唤醒词。
- v1 不直接控制用户电脑、文件或应用，除非后续明确设计权限系统。
- v1 不保存原始 API Key 明文到文档、日志或测试。
- v1 不做账号体系和云同步。
- v1 不复制外部项目的源码、品牌、图片、Logo 或受许可约束的原创视觉资产。
