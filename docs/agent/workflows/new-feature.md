# Workflow：新增功能

## 目标

新增功能时，agent 必须先确认需求、边界和验收标准，再实现代码。

## 步骤

1. 阅读 `AGENTS.md`、`00-index.md`、项目概览、架构、技术栈和质量规则。
2. 找到相关模块文档；不存在时先指出缺口，并按 `_template.md` 补齐或建议补齐。
3. 查找现有实现、相似功能、测试和命名风格。
4. 明确功能输入、输出、错误路径、配置需求和验收标准。
5. 选择最小实现路径，避免提前扩大架构。
6. 实现功能，并补充必要测试。
7. 运行相关验证。
8. 若新增公共接口、配置、模块或架构边界，更新 `docs/agent/`。
9. 按 `task-report-template.md` 汇报。

## 禁止事项

- 不允许跳过现有实现搜索直接写新代码。
- 不允许为了新功能绕过模块边界。
- 不允许引入未记录的新依赖。
- 不允许把临时实验代码留在主路径。
