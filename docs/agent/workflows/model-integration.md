# Workflow：模型接入

## 目标

模型接入必须保证 API Key 安全、供应商可替换、流式事件稳定、错误可理解。

## 步骤

1. 阅读 `modules/ai-runtime.md`、`modules/security.md` 和技术栈文档。
2. 确认供应商、模型名称、认证方式、流式协议和费用风险。
3. 设计或更新供应商适配层，不让供应商 shape 泄露到 UI。
4. 实现 API Key 配置、验证、请求、取消、超时和错误归一化。
5. 补充上下文组装、流式解析和失败路径测试。
6. 更新 `04-decisions.md` 和相关模块文档。

## 禁止事项

- 不允许在 renderer 中保存或拼接 API Key。
- 不允许把真实密钥写入 `.env.example`、测试、日志或文档。
- 不允许只实现成功路径。
