# 模块：视觉系统

## 模块职责

视觉系统负责 Three.js/WebGL 粒子场、弧核动画、状态过渡、音量响应和沉浸式背景。

不负责模型请求、语音识别、会话存储或业务状态推断。

## 公共接口

- 输入：`idle`、`listening`、`transcribing`、`thinking`、`speaking`、`muted`、`error` 等状态。
- 输入：可选音量、流式 token 节奏、错误级别等轻量信号。
- 输出：仅限视觉事件或性能状态，不直接修改聊天业务数据。
- 当前 renderer 使用共享 `VisualSignal` 将音频频段、token 脉冲、错误闪断、思考强度和播报强度传入 `ParticleCore`。

## 数据与状态

- 管理 WebGL renderer、scene、camera、geometry、materials、animation frame 和资源释放。
- 必须支持窗口 resize、暂停、恢复和低性能降级。
- 当前 `ParticleCore` 已按设备像素比和 CPU 并发数选择 `low`、`medium`、`high` 性能档位，调整粒子数量、像素比、轨道环数量和高级光效。
- 窗口或页面失焦后，视觉动画必须降到低频刷新；恢复可见后再回到 `requestAnimationFrame`。
- 当前 `ParticleCore` 已支持启动能量、流式 token 脉冲、错误红色闪断、思考聚合、播报波形、音频低/中/高频和节奏响应。
- 当前麦克风采样会从 `AnalyserNode` 同时读取 RMS 音量和频谱分段，低频影响粒子大小与核心亮度，中频影响轨道速度，高频和节奏影响波形与细碎脉冲。
- 当前 `ParticleCore` 默认以更大的近景弦核显示，支持鼠标拖拽旋转和滚轮缩放；左侧会话抽屉打开时会传入侧栏状态，让弦核产生轻微 3D 视角偏转。

## 边界规则

- 视觉状态必须由业务状态驱动，不制造虚假模型进度。
- 不复制外部项目的原创视觉资产或源码。
- 不使用会导致输入卡顿的持续高成本动画。
- HUD 只能展示模型、会话、记忆、token 和输入信号等状态，不应遮挡对话与输入，也不应伪造业务进度。
- HUD 默认放在侧边状态栏，不应占据弦核中心视觉区域。

## 验证要求

- 检查画布非空白。
- 检查桌面和窄窗口尺寸。
- 检查状态切换可见。
- 检查鼠标拖拽旋转、滚轮缩放和左侧抽屉打开时的弦核偏转不会影响输入、设置、记忆和对话操作。
- 检查资源释放和重复进入页面不泄漏。
- 检查 HUD 在桌面和窄窗口下不遮挡输入、设置、记忆和对话。
