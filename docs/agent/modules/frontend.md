# 模块：Renderer Frontend

## 模块职责

Renderer 负责 React 界面、对话体验、设置、历史、记忆管理、状态展示和用户交互。

不负责直接访问 Node API、文件系统、数据库、系统密钥或模型 API Key。

## 公共接口

- 只通过 preload 暴露的 typed `window.arcMind` 获取能力。
- 与视觉系统通过状态和事件通信，不直接操纵底层模型请求。

## 数据与状态

- 管理页面级状态、表单状态、聊天渲染状态和用户操作反馈。
- 模型流式内容应可停止、重试和错误恢复。
- 当前 renderer 通过 `window.arcMind.chat` 发送消息、订阅 token、停止生成；通过 `window.arcMind.settings` 保存/测试模型配置，页面只展示是否已配置 API Key。
- 设置面板的“测试连接”应传入当前表单草稿；用户不必先点击保存才能测试刚输入的 API Key。
- 当前会话历史使用左侧边缘触发抽屉，默认隐藏以减少对弦核视觉的遮挡；抽屉打开状态会传给 `ParticleCore` 用于轻微视角偏转。
- 弦核 HUD 使用中文标签，并放置在右侧状态栏，避免覆盖中心动态效果。

## 边界规则

- UI 必须遵守 `design-system.md`。
- 不在页面中显示 API Key、私密路径或不可删除的记忆内容。
- 输入、发送、停止、重试、静音和录音按钮必须有明确状态。

## 验证要求

- 组件测试或交互测试。
- 窄窗口和桌面窗口手动检查。
- 视觉层叠检查：粒子不得遮挡对话和按钮。
