# 评审规则

代码评审以发现风险为第一目标，优先级高于总结改动。

## 评审关注点

- 是否破坏 Electron main、preload、renderer、AI Runtime、Voice Runtime、存储和视觉系统的边界。
- 是否把 API Key、token、私密对话、语音样本或本地路径写入不该出现的位置。
- 流式模型输出、取消、重试、超时和错误是否处理清楚。
- 录音、播放和 WebGL 渲染是否有释放逻辑，避免资源泄漏。
- 用户可见错误是否说明发生了什么和用户能做什么。
- UI 是否符合 `design-system.md`，没有文字重叠、按钮挤压和无意义装饰。
- 新增公共接口、配置或依赖是否同步更新文档和测试。

## 输出格式

评审结论优先列出问题，按严重程度排序，并给出文件和行号。没有发现问题时，明确说明“未发现阻塞问题”，并补充剩余测试缺口或风险。
