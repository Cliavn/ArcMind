# 界面风格规范

本文件定义 ArcMind 的长期界面风格。任何视觉、交互、动画、前端组件或粒子系统改动，都必须先阅读并遵守本规范。

## 风格命名

项目界面采用 **ArcMind Core Console** 风格，中文定位为 **弧核智能终端**。

它不是普通客服聊天框，不是营销型 landing page，也不是单纯炫技 HUD。它应该像一个本地 AI 终端：沉浸、克制、清晰、有回应感，让用户一眼感到 AI 正在聆听、思考或回答。

## 已选视觉基线

默认视觉方向采用 **深色粒子核心 + 可读对话界面 + 小面积 HUD 信息层**。

- **粒子核心是识别点**：全屏 WebGL/Canvas 背景呈现弧形粒子、能量核心、声波脉冲和状态变化。
- **对话可读是底线**：聊天内容、输入框、设置和错误反馈必须清晰可读，不被视觉层遮挡。
- **状态驱动优先**：待机、聆听、思考、回答、错误、静音必须有稳定且可区分的视觉表现。
- **克制电影感**：可以有扫描线、微光、粒子轨道和透明面板，但不能让界面变成噪声堆叠。

## 设计关键词

- **弧核**：环形、轨道、核心、脉冲是主要视觉语言。
- **私人助手**：语气温和、反馈及时、状态清楚。
- **沉浸但可控**：第一屏有氛围，但设置、历史和错误处理保持桌面软件的可靠性。
- **状态同步**：视觉层必须响应真实业务状态，不自行制造虚假进度。
- **隐私可信**：记忆、历史、麦克风、模型配置必须让用户知道在哪里、怎么关、怎么删。

## 布局规则

- 首屏直接呈现可聊天体验，不做营销 hero。
- 视觉层全屏铺底；对话层、输入层和状态层作为稳定浮层。
- 聊天内容区域必须有足够对比度和可读宽度，不被粒子高亮干扰。
- 设置、历史、记忆管理优先使用侧栏、抽屉或独立面板，不打断主对话。
- HUD 信息层只展示模型状态、输入信号、会话摘要、记忆数量和流式节奏等短状态，不承担说明文档或操作主入口。
- HUD 信息层默认靠侧边排布，不覆盖弦核中心动态效果；左侧历史入口可采用边缘触发抽屉。
- 固定格式元素必须有稳定尺寸，流式文字、状态标签和按钮变化不得造成明显布局跳动。
- 滚动容器必须统一定制滚动条样式，避免回退到浏览器原生默认外观。
- 不使用卡片套卡片；需要层级时使用边界、透明度、分割线和空间组织。

## 色彩系统

默认色彩应避免被单一紫色、深蓝或青色统治。建议语义方向：

- `space`：近黑背景，用于主画布。
- `panel`：半透明深色面板，用于对话和设置。
- `ink`：高亮正文。
- `muted`：次级说明和时间。
- `arc-cyan`：核心粒子和聆听状态。
- `arc-gold`：思考、启动和能量过渡。
- `arc-green`：成功、可用、记忆已保存。
- `arc-red`：错误、断连、危险操作。
- `line`：弱边界和 HUD 线条。

使用规则：

- 强发光色只用于核心状态、焦点和小面积反馈。
- 错误态必须明显，但不要用大面积红色覆盖整个界面。
- 状态色必须配合文字、图标或动画模式，不只依赖颜色。
- 避免装饰性渐变光球和无业务含义的大面积背景块。

## 字体与文字

- 默认字体使用系统无衬线字体，中文优先清晰可读。
- 聊天正文建议 `14px` 到 `16px`。
- HUD 标签和元信息建议 `11px` 到 `12px`。
- 页面标题和对话角色名保持克制，不使用夸张 hero 级字体。
- 字距保持默认，不使用负字距。
- 页面内不写功能说明书、快捷键教学或设计解释。

## 图标与组件

- 操作按钮优先使用 `lucide-react` 图标，并提供 `title` 或 tooltip。
- 图标按钮用于麦克风、静音、发送、停止、设置、历史等高频动作。
- 按钮圆角默认不超过 `8px`，除非是状态环、圆形麦克风按钮或视觉核心控件。
- 输入框必须支持发送、停止生成、禁用、错误和 loading 状态。
- 记忆与隐私相关操作必须有明确文字，不只用图标。

## 动效规则

- 粒子动画服务状态反馈，不制造虚假计算。
- 常规 UI 动效建议 `120ms` 到 `220ms`。
- 粒子状态过渡可使用 `300ms` 到 `900ms`，但不影响输入响应。
- 聆听状态可响应音量；思考状态可呈现聚合或轨道加速；回答状态可呈现脉冲扩散。
- 流式 token 可触发短促粒子脉冲；错误可触发短促红色闪断；语音输入和播报可用低/中/高频与节奏分别影响粒子大小、轨道速度、核心亮度和波形。
- WebGL 场景必须在低性能设备上有降级策略。
- loading、流式回复和 TTS 播放不得造成布局跳动。

## 页面级验收

任何 UI 改动完成前，必须检查：

- 是否一眼能看出 AI 当前状态。
- 聊天内容和输入是否清晰可读。
- 粒子视觉是否没有遮挡文本和按钮。
- 常见窗口尺寸下是否没有文字重叠或按钮挤压。
- 麦克风、静音、停止生成、错误重试是否有清楚状态。
- 视觉层是否能在性能较弱时降级或关闭。
